blog规范

发表于 2024-08-05 分类于 tools
本文字数： 639 阅读时长 ≈ 1 分钟

blog基于Hexo+next搭建，部署于GitHub Pages，采用Typora+Markdown编写，使用阿里云图床。

标题

  1. blog使用二级标题，主体内容使用四、五级标题分级，一般使用二级标题、四级标题即可。

  2. 博客标题应采用XXX的运用/实现（偏正短语），例如：基于Java背包算法的实现。

    注意：字数不宜超过15字，英文单词视为一字。

标签

​ 一级标签表示内容分类，二级标签表示内容关键词，每级标签数量为2、3个。

插图

  1. 本地必须进行图片压缩，以节省服务器空间。

  2. 命名格式为“所属标题_插图内容关键词”，例如：插图_压缩软件使用说明。

  3. 每张插图需给出明确的上下文说明，数量要与文章篇幅相符，非必要不贴图。

  4. 插图应按67%大小进行缩放，长宽比为16 : 9。

代码

​ 代码需完整且能够编译运行，使用注释说明，良好代码格式，如若涉及具体算法，需说明其时间复杂度与空间复杂度。

发布

  1. 时间安排，计划以周为单位，每周发布一篇博客。

  2. 内容为某要点的说明，例如：Mysql窗口函数的使用。

  3. 博客发布详见教程。

备份

成功发布博客后，分别及时在Gitee、百度云、蓝奏云平台进行数据备份。

  1. Gitee采用git管理。

  2. 百度云、蓝奏云使用传统的文件上传方式。
